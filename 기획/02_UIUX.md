> 해결 대상 피드백: **"게임 사용법도 모르겠고 뭐가 뭔지도 모르겠다."**

## 0. 핵심 진단 (현재 OnGUI HUD의 5대 결함)
| # | 현재 문제 | 이 문서의 해결책 |
|---|---|---|
| 1 | 첫 화면에 타이틀/스테이지/튜토리얼 없음 → 곧장 게임판이 뜸 | §1 화면 흐름 + §2 인터랙티브 튜토리얼 |
| 2 | "어디에 짓는지" 파란 타일이 카메라 각도/조명에 묻혀 안 보임 | §4 설치 타일 시각언어(펄스 링·아이콘·테두리) |
| 3 | 타워 선택/업그레이드/판매 UI 전무 | §3 선택 타워 패널 + §5 인터랙션 |
| 4 | 다음 웨이브에 뭐가 오는지 정보 없음 | §3 다음 웨이브 미리보기 |
| 5 | 피드백(주스) 없음 → 무엇이 일어나는지 체감 안 됨 | §6 피드백/주스 + 사운드 매핑 |

설계 원칙: **(a) 한 번에 하나씩 가르친다, (b) 다음 행동을 항상 화면이 가리킨다, (c) 모든 상호작용에 시청각 반응을 준다.**

## 1. 화면 흐름도

```
                         ┌─────────────┐
                 ┌──────▶│   타이틀     │◀───────┐
                 │       └──────┬──────┘         │
                 │              │ [시작]          │ [메인으로]
                 │              ▼                 │
                 │       ┌─────────────┐          │
            [뒤로]│       │ 스테이지 선택 │          │
                 │       │  (월드맵)    │          │
                 │       └──────┬──────┘          │
                 │   (첫 진입=튜토리얼 강제)         │
                 │              ▼                  │
                 │       ┌─────────────┐           │
                 └───────┤   인게임     ├─[일시정지]─┤
                         │  (HUD+맵)   │  팝업      │
                         └──────┬──────┘           │
                          승/패  ▼                  │
                         ┌─────────────┐            │
                         │ 결과(별점)   ├────────────┘
                         └─────────────┘
                          [재도전] [다음] [선택으로]

   오버레이(어느 화면에서나): 설정 ⚙ · 도움말 ? · 일시정지 ⏸
```

| 화면 | 목적 | 핵심 요소 | 전환 |
|---|---|---|---|
| **타이틀** | 게임 정체성·진입점 | 로고, [시작]/[이어하기], ⚙설정, ?도움말, 음소거 | [시작]→스테이지 선택 |
| **스테이지 선택** | 진행도·다음 도전 제시 | 월드맵 노드(잠금/별점), 스테이지 카드(미리보기·난이도·최고기록) | 노드 탭→인게임 / 뒤로→타이틀 |
| **인게임** | 실제 플레이 | 맵, HUD(§3), 건설 팔레트, 타워 패널, 일시정지 | 승/패→결과 / ⏸→일시정지 |
| **일시정지(팝업)** | 중단·점검 | [계속] [재시작] [설정] [스테이지 선택], 게임 정지 | [계속]→인게임 |
| **결과** | 성취 확인·재동기 | 승/패 타이틀, ★1~3, 남은 라이프·처치수, [재도전][다음][선택] | 각 버튼 분기 |
| **설정(팝업)** | 사운드·도움말 | BGM/효과음 볼륨, 배속 기본값, 언어(고정 한국어), 튜토리얼 다시보기 | [닫기] |
| **도움말(팝업)** | 규칙 요약 | 1장짜리 카드(타워설치/골드/라이프/배속 아이콘 설명) | [닫기] |

> MVP 우선순위: **인게임 HUD 재설계 + 설치 타일 + 튜토리얼**이 1순위. 타이틀/스테이지 선택은 단순 버전(스테이지 1개+튜토리얼 노드)으로 시작해도 됨.

## 2. 첫 플레이 온보딩 / 인터랙티브 튜토리얼

스테이지 0 = **튜토리얼 전용**. 짧은 직선 경로 + 슬롯 1~2개 + 약한 적 소수. 첫 진입 시 자동 진입(설정에서 다시보기 가능). **스텝 게이팅**: 요구 행동을 끝내기 전엔 다음 단계로 못 넘어가고, 화면도 그 행동만 가리킨다.

### 가이드 UI 구성요소
- **딤(dim) 마스크**: 화면 전체를 반투명 검정으로 덮고, 가르칠 대상(타일/버튼)만 구멍 뚫어 강조(spotlight).
- **포인터**: 대상으로 향하는 손가락👆(터치) / 화살표▶(마우스) 아이콘, 위아래 살짝 바운스.
- **말풍선**: 대상 옆에 Kenney UI Pack 패널. 한국어 카피 + [확인]/자동 진행.
- **진행 표시**: 우상단 "튜토리얼 1/5".

### 단계 (스텝 게이팅)
| 단계 | 화면이 하는 일 | 게이팅(이 행동 전까지 잠금) | 말풍선 카피(예시) |
|---|---|---|---|
| 1. 환영 | 맵 전체 보여주고 경로를 화살표로 훑음 | [확인] 탭 | "적이 이 길을 따라 우리 기지로 쳐들어와요. 막아봅시다!" |
| 2. 타일 강조 | 설치 타일 1개만 펄스 링+손가락 가리킴, 나머지 딤 | **그 타일 탭** 전까지 | "반짝이는 칸을 눌러 타워를 세워요. (50골드)" |
| 3. 타워 확정 | 건설 미니 메뉴(타워 1종) 열림, 손가락이 [건설] 가리킴 | **건설 버튼 탭** 전까지 | "이 타워는 가까운 적을 자동으로 쏴요." |
| 4. 웨이브 시작 | 하단 [웨이브 시작] 버튼만 강조 | **버튼 탭** 전까지 | "준비됐으면 시작! 적이 나와요." |
| 5. 전투 관전 | 첫 적 처치 순간 골드 팝업+화살표로 골드 카운터 강조 | 적 1마리 처치 시 자동 | "처치하면 골드를 벌어요. 골드로 타워를 더 지어요." |
| 6. 라이프 경고 | 적이 기지 근접 시 라이프 아이콘 흔들고 강조(연출용 1마리 통과 허용) | 자동 | "적이 기지에 닿으면 라이프가 줄어요. 0이 되면 패배!" |
| 7. 완료 | 웨이브 클리어 → 결과 미니 | [계속] | "끝! 이제 진짜 스테이지로 가볼까요?" |

가이드 원칙:
- **한 번에 한 동작**. 동시에 두 곳을 가리키지 않는다.
- **틀린 곳 탭하면** 살짝 흔들고("거기 말고 여기예요!") 올바른 대상을 다시 강조.
- **건너뛰기**: 우상단 [건너뛰기] 항상 노출(재방문 유저 배려). 단 첫 1회는 약하게 표시.

## 3. 인게임 HUD 재설계

OnGUI → **Unity UI(Canvas, Screen Space - Overlay) + Kenney UI Pack 9-slice 패널**로 전환. 모바일 가독성을 위해 터치 타깃 최소 **48×48px**, 안전영역(safe-area) 패딩.

### 레이아웃 (3영역)
```
┌──────────────────────────── 상단 바 ────────────────────────────┐
│  💰 골드 120   ❤ 라이프 10   🌊 웨이브 2/3      [⏸] [▶ 1x] [⚙] │
├──────────────────────────────────────────────────────────────┤
│                                                                │
│                       (3D 게임 보드 영역)                        │
│                                                                │
│   ┌─ 선택 타워 패널(타워 선택 시만) ─┐                            │
│   │ 🔫 화살탑 Lv.1                  │     ┌─ 다음 웨이브 ─┐       │
│   │ 데미지 20 · 사거리 ●            │     │ 🛸×6  🛸⚡×4   │       │
│   │ [⬆ 업그레이드 30] [💲 판매 25] │     └───────────────┘       │
│   └────────────────────────────────┘                          │
├──────────────────────────── 하단 바 ────────────────────────────┤
│  [🔫 화살탑 50]  [💥 광역탑 80]            [ ▶ 웨이브 시작 ]      │
└──────────────────────────────────────────────────────────────┘
```

### 요소별 사양 / 우선순위
| 영역 | 요소 | 내용 | 우선순위 |
|---|---|---|---|
| 상단 좌 | 자원 칩 | 골드/라이프/웨이브. 값 변할 때 펀치 스케일+색 점멸(골드↑녹색, 라이프↓빨강) | P0 |
| 상단 우 | 컨트롤 | 일시정지 ⏸, 배속 토글(1x↔2x↔3x), 설정 ⚙ | P0(배속 P1) |
| 하단 좌 | **건설 팔레트** | 타워 카드(아이콘+이름+가격). 골드 부족 시 회색+가격 빨강. 탭→건설 모드 | P0 |
| 하단 우 | **웨이브 시작** | Ready 상태에서만 활성. 큰 버튼. 클릭 시 카운트다운 살짝 | P0 |
| 우측 | **다음 웨이브 미리보기** | 다음 적 종류 아이콘×수량. 호버/탭으로 상세(체력·속도) | P1 |
| 중앙 팝업 | **선택 타워 패널** | 이름·레벨·데미지·사거리(시각 원), [업그레이드(가격)] [판매(환급액)] | P0 |

규칙:
- 패널은 **타워/슬롯 선택 시에만** 등장(평소 화면 깨끗하게).
- 빈 곳 탭 = 선택 해제(패널 닫힘, 사거리 원 사라짐).
- 모바일에서 선택 타워 패널은 화면 하단에 붙는 **바텀시트**로 변형(손가락 닿기 쉽게).

## 4. 설치 가능 타일 명확화 (핵심 문제 직접 해결)

현재: 파란 타일이 카메라 부감+조명에 묻혀 "안 보임". → **타일 색만으로 의존하지 않고, 항상 보이는 수직 신호 + 상태별 시각언어**를 더한다.

### 상태별 시각 언어
| 상태 | 색/형태 | 추가 신호 | 비고 |
|---|---|---|---|
| **설치 가능** | 청록(#36C5FF) 타일 + **펄스 링**(바닥에 숨쉬듯 커졌다 작아지는 원) | 타일 위로 살짝 떠있는 ⊕(플러스) 아이콘 빌보드 | 건설 모드 아닐 때도 은은히 표시(첫 학습용), 건설 모드 진입 시 강하게 |
| **설치 가능·호버/선택** | 밝은 시안 + 외곽 굵은 테두리 글로우 | 손가락/커서 위치에 **고스트 타워**(반투명 프리뷰) + **사거리 원** | 마우스 호버 또는 터치 후보 위치 |
| **점유됨(타워 있음)** | 타일 신호 끔, 타워만 표시 | 탭하면 선택 타워 패널 | 펄스 링 없음 |
| **설치 불가**(경로/장애물) | 회색·신호 없음 / 건설 모드 중엔 ✕ 빨강 표시 | 그 위 드롭 시 빨강 점멸+거부음 | 경로 타일은 항상 흙길 텍스처로 구분 |

핵심 처방(현재 "안 보임" 직격):
1. **수직 빌보드 아이콘 ⊕** — 카메라 각도와 무관하게 항상 정면을 봄(바닥 색에 안 묻힘).
2. **펄스(숨쉬기) 애니메이션** — 정지된 파란 사각형보다 움직이는 신호가 눈에 먼저 띔.
3. **건설 모드 대비** — 팔레트에서 타워를 고르면, 설치 가능 타일이 **전체 동시에 환하게** 부각(어디든 지을 수 있는지 한눈에).
4. **에미시브 글로우 유지** — 조명 영향 적게(이미 코드에 `_EMISSION` 사용 중 → 색만 청록으로, 강도 상향).

### 사거리 미리보기 원
- 건설 모드에서 후보 타일 위에 **반투명 디스크**(타워 사거리 = `cell*3.5f`)를 바닥에 깔아 보여줌.
- 이미 설치된 타워 선택 시에도 같은 원 표시(어디까지 닿는지 즉시 이해).

### 건설 모드 진입/취소 흐름
```
팔레트 타워 카드 탭
   → [건설 모드 ON]  · 가능 타일 전부 환하게 · 손가락 따라 고스트+사거리 원
   → 가능 타일 탭/드롭   → 골드 차감 + 설치 + 설치 이펙트 + 사운드 → [건설 모드 OFF]
   → 불가 위치 탭        → 거부 점멸+거부음 (모드 유지)
   → 빈 곳/취소(✕)/ESC   → [건설 모드 OFF] (차감 없음)
```

## 5. 타워 설치·선택 인터랙션 (마우스 & 터치)

공통 입력 레이어로 추상화(기획서 §7). "포인터 다운/업/이동"을 마우스·터치 공용으로 처리.

### 마우스(PC)
| 동작 | 결과 |
|---|---|
| 팔레트 카드 클릭 | 건설 모드 진입, 커서에 고스트 타워 |
| 가능 타일 위 호버 | 타일 강조 + 고스트 + 사거리 원 |
| 가능 타일 클릭 | 설치 확정 |
| 설치된 타워 클릭 | 선택 → 타워 패널 + 사거리 원 |
| 빈 곳/우클릭/ESC | 선택 해제 · 건설 취소 |

### 터치(모바일)
| 동작 | 결과 |
|---|---|
| 팔레트 카드 탭 | 건설 모드 진입(가능 타일 전체 강조) |
| 가능 타일 **탭** | 그 자리에 즉시 설치(가장 단순한 1탭 설치) |
| (대안) 카드에서 **드래그** | 손가락 따라 고스트, 가능 타일 위에서 **놓으면** 설치 / 불가면 취소 |
| 설치된 타워 **탭** | 선택 → 바텀시트 패널 + 사거리 원 |
| 타워 **롱프레스(0.5s)** | 빠른 메뉴(업그레이드/판매) 바로 노출(선택사항) |
| 빈 곳 탭 | 선택/건설 해제 |

오배치 방지 & 취소:
- 불가 위치엔 **놓이지 않음**(드롭 시 원위치 복귀 + 거부음). 골드는 **확정 시점에만** 차감.
- 1탭 설치 모드에서 실수 방지: 설치 직후 **3초 즉시 환불 [되돌리기]** 토스트(전액 환급).
- 골드 부족 시 카드 자체가 비활성 → 애초에 건설 모드 진입 불가(가격 빨강 흔들림 + 거부음).

## 6. 피드백 / 주스(juice) — 과하지 않게

목표: "지금 뭔가 일어나고 있다"를 즉시 체감. 모든 이펙트는 **짧고(≤0.4s)·작게**. WebGL 성능 위해 풀링·간소 파티클.

| 사건 | 시각 | 사운드(Kenney 팩 → 파일 예시) | 비고 |
|---|---|---|---|
| 타워 설치 | 바닥 먼지 퍼프 + 타워 살짝 솟구침(스케일 팝) | interface/ui → `click`, impact → `thud` | 확정 피드백 |
| 타워 발사 | 총구 섬광 + 투사체 트레일 | sci-fi → `laser` 짧게(연사 시 볼륨↓·간헐) | 연사 사운드 스팸 주의 |
| 적 피격 | 적 흰색 깜빡(히트 플래시) + 작은 스파크 | impact → `impactSoft` | 데미지 전달 |
| **데미지 숫자** | 적 위로 작은 숫자 떠오르며 페이드업 | (무음) | 기본 ON, 설정서 OFF 가능 |
| 적 처치 | 파편 퍼프 + 살짝 축소 후 소멸 | sci-fi → `explosion` 작게 | |
| **골드 획득 팝업** | 처치 위치에 "+12" 금색, 골드 카운터로 빨려 들어감 | rpg → `coin` | 보상감 핵심 |
| 라이프 감소 | 화면 가장자리 **빨강 비네트** 한 번 번쩍 + 라이프칩 흔들림 | interface → `error`/`buzz` | 위기 경고 |
| 웨이브 시작 | 상단에 "웨이브 N" 배너 슬라이드 + 카운트다운 | music-jingles → 짧은 `start` | |
| 승리 | 별점 ★ 하나씩 팝 + 팡파레 | music-jingles → `win` | |
| 패배 | 화면 회색 페이드 + 다운 사운드 | music-jingles → `lose` | |
| 버튼/카드 | 호버 살짝 밝게, 탭 시 눌림 | ui → `tick` | 전역 |

전역: BGM 1트랙(음량 작게, 음소거 토글), 효과음/배속 따로. 첫 사용자 제스처(첫 탭) 후 오디오 시작(브라우저 autoplay 정책).

## 8. 다른 팀 의존성 & 열린 질문

### 아키텍처팀에 요청(UI가 구독할 상태/이벤트)
UI는 `GameManager`에서 데이터를 직접 읽지 않고 **이벤트/상태 구독**으로 분리하길 권장:
- 상태값 노출: `Gold`, `Lives`, `WaveIndex/WaveTotal`, `GameState(Ready/InWave/Won/Lost)`.
- 이벤트: `OnGoldChanged(delta, worldPos)`, `OnLivesChanged(delta)`, `OnWaveStart(WavePreview)`, `OnWaveEnd`, `OnEnemyHit(pos,dmg)`, `OnEnemyKilled(pos,goldGained)`, `OnEnemyReachedBase`, `OnTowerBuilt/Upgraded/Sold(slot)`, `OnTowerSelected(towerInfo)`.
- **다음 웨이브 미리보기 데이터**: 현재 `WaveEntry{basic,fast}`는 내부 struct → UI가 읽을 수 있게 `WavePreview{ (적종류, 수량)[] }` 형태로 공개 필요.
- **타워 메타** 조회 API: `TowerInfo{ name, icon, cost, damage, range, level, upgradeCost, sellRefund }`.
- 슬롯 상태 조회: 슬롯별 `Buildable/Occupied/Blocked` + 사거리 원 렌더용 range.
- 입력 추상화: 포인터(다운/업/이동) 공용 레이어(마우스+터치). 건설 모드 상태머신은 UI 측이 소유하되 설치 확정만 GameManager 호출.

### 레벨기획팀에 의존(데이터)
- **타워/적 로스터**: 아이콘 이미지(Kenney에서 매핑), 한국어 이름·1줄 설명, 수치(데미지/사거리/체력/속도/가격/환급/업그레이드 비용·효과).
- **웨이브 테이블**: 웨이브별 적 구성/타이밍(미리보기 표시용).
- **스테이지 정의**: 스테이지 수, 잠금/해금 조건, 별점 기준값(라이프 임계).
- **튜토리얼 스테이지(0)** 전용 레벨(짧은 경로·슬롯 1~2·약한 적) 정의.

### 열린 질문(결정 필요)
1. **웨이브 진행 방식**: 수동 [시작] 유지 vs 자동(다음 웨이브 자동 카운트다운, 미리 시작 시 보너스 골드)? — 온보딩 난이도에 영향.
2. **타워 종류**: 현재 코드는 1종만 건설. MVP에 광역탑(2종) 포함 확정? 팔레트/패널 설계는 2종 가정.
3. **업그레이드/판매**: 코드 미구현. MVP 포함 여부 + 단계 수(예: Lv1~3) 결정 필요.
4. **스테이지/월드맵**: MVP는 단일 스테이지+튜토리얼만? 월드맵은 다음 단계로 미룰지.
5. **배속**: 2x/3x 제공 범위(WebGL 성능 영향) — 물리/스폰 타이밍 스케일 검증 필요.
6. **저장**: 진행도/별점 저장(localStorage/Firebase)? "이어하기"·월드맵 잠금에 필요.
7. **UI 기술**: Unity UGUI(Canvas) vs UI Toolkit. 권장 UGUI(Kenney 9-slice·터치 다루기 쉬움, 빠른 반복). 한글 폰트는 TextMeshPro에 Pretendard SDF 폰트에셋 생성 필요(현재 동적 Font와 별개).
8. **사거리 원/펄스 링** 렌더 방식: 바닥 데칼 vs 메시 디스크 vs 셰이더 — WebGL 호환 우선.
