> 작성 2026-06-23. 3개 팀(아키텍처/UI·UX/레벨·난이도) 병렬 설계를 통합한 상위 문서.
> 상세는 각 문서 참조: [01_아키텍처](01_아키텍처.md) · [02_UIUX](02_UIUX.md) · [03_레벨난이도](03_레벨난이도.md)

## 0. 왜 다시 짜는가
현재 빌드(https://wiitower.web.app)는 **그레이박스 기술 데모**다. 모든 게 `GameManager.cs` 한 파일에 하드코딩(맵1·타워1·적2·웨이브3), UI는 OnGUI, 타이틀·스테이지선택·튜토리얼·업그레이드·저장·사운드 전무.
→ 오너 피드백: **"사용법도 모르겠고 뭐가 뭔지도 모르겠다."**
원인은 두 가지: (1) 처음 보는 사람을 가르치는 **온보딩/UX가 없음**, (2) 콘텐츠가 **하드코딩이라 확장 불가**. 둘 다 구조 문제라 부분 수정이 아니라 재설계로 간다.

## 1. 비전 / 세계관
**"마지막 지상 방어선의 사령관이 되어, 끝없이 내려오는 외계 UFO 침공을 지상 포탑으로 격추하라."**
- 톤: 밝은 SF·캐주얼 디오라마(Kenney 저폴리 "장난감 전장"). 명료함 + 손맛.
- 전선이 **초원 → 설원 → 크리스탈 균열**로 이동 = 3개 월드 = Kenney 타일 바이옴 3종 (추가 아트 0원).

## 3. 통합 아키텍처 (요약)
DI 프레임워크 없이 **정적 ServiceLocator + GameEvents(이벤트 버스)** 로 느슨 결합. 씬 1개 + Bootstrap 유지.
```
Core/        Bootstrap, ServiceLocator, GameEvents, AppFSM(Title/StageSelect/InGame/Result)
Content/     ContentDB(JSON→Def 적재), TowerDef/EnemyDef/WaveDef/StageDef
Map/         MapGrid(ASCII), PathService(BFS 경로), SlotService(설치 타일)
Battle/      BattleFSM, WaveRunner, Spawner(풀링), Economy
Units/       Enemy, Tower, Projectile (+ ObjectPool)
Combat/      Targeting, Effects(slow/armor/splash/aura/...)
Input/       PointerInput (마우스+터치 공용)
UI/          (uGUI) HUD, BuildPalette, TowerPanel, WavePreview, Title, StageSelect, Result, Tutorial
Audio/       SfxService (Kenney)
Save/        ProgressStore (localStorage)
```
- UI는 **GameEvents 구독만**으로 표시, 버튼은 intent 이벤트만 발행(로직 직접 호출 금지).
- 적/발사체 **오브젝트 풀링**으로 WebGL GC 스파이크 방지.
- 난이도는 **단일 모듈의 공식**으로 계산(매직넘버 제거).

### 데이터 스키마 (4종)
- **TowerDef**: id, 이름, 모델키, 티어별(데미지/사거리/공속/코스트/효과), 상성태그
- **EnemyDef**: id, 이름, 모델키, HP/속도/태그(ground/flying/fast/armored/swarm/boss)/보상
- **WaveDef**: SpawnGroup[](적id·수량·간격·딜레이)
- **StageDef**: 월드, ASCII맵, 시작골드/라이프, 웨이브[], 별점기준, 해금조건
> 맵 예시: `S`=스폰 `G`=기지 `#`=경로 `.`=잔디 `o`=설치슬롯 → PathService가 S→G 추적, 연결성 검증.

## 5. 스테이지 / 난이도
- **3월드 × 5스테이지 = 15개** (보스 St5/10/15). 신규 요소는 스테이지당 **1개씩만** 도입(학습 곡선).
  - St1 pulse/scout(튜토) → St2 mortar/스웜 → St3 cryo/고속+업글개방 → St4 탱커 → **St5 보스** → St6 flak/비행 → St7 장갑 → St8 분기경로 → St9 pylon/보호막 → **St10 보스** → St11 점멸 → … → **St15 최종보스**
- 난이도 공식(월드W·스테이지s·웨이브w):
  - `enemyHP = baseHP × (1+0.7(W−1)) × (1+0.12(s−1)) × (1+0.06(w−1))`
  - `waveCount = 5 + floor((전체스테이지−1)/3)` (보스 +1)
  - `startGold = 120 + 10(S−1)`, `waveClearBonus = 15 + 5w`, 판매환불 70%
  - **별점**: 3성=무실점 · 2성=라이프손실≤5 · 1성=클리어. 월드 해금=누적 별 게이트.

## 6. UX / 온보딩 (요약)
- **화면 흐름**: 타이틀 → 스테이지 선택(월드맵) → 인게임 → 결과(★1~3). 오버레이: 일시정지·설정·도움말. 첫 진입은 **튜토리얼 스테이지 강제**.
- **튜토리얼(스텝 게이팅 7단계)**: 환영 → 타일 강조(탭 전까지 잠금) → 타워 확정 → 웨이브 시작 → 골드 학습 → 라이프 경고 → 완료. 딤 마스크 spotlight + 손가락👆 포인터 + 한국어 말풍선.
- **"설치 타일 안 보임" 해결(현 핵심 결함)**: ① 카메라 무관 정면 빌보드 ⊕아이콘 ② 펄스(숨쉬기) 링 애니메이션 ③ 건설 모드 진입 시 가능 타일 전체 동시 부각 ④ 에미시브 청록 글로우. 상태 시각언어: 가능/호버(고스트+사거리원)/점유/불가✕.
- **HUD**: 상단 자원칩+컨트롤, 우측 다음 웨이브 미리보기(🛸×N), 하단 건설 팔레트+웨이브 시작, 선택 타워 패널(업글/판매).

## 7. 통합 로드맵 (각 단계 끝마다 WebGL 빌드·배포로 회귀 확인)
**A. 기반 (수직 슬라이스)** — "제대로 게임답다"를 빠르게
- **M0** 골격: 폴더/네임스페이스, ServiceLocator+GameEvents, Bootstrap(현 GameManager 진입점 이동). 기존 동작 유지.
- **M1** 데이터화: Newtonsoft 도입, ContentDB, towers/enemies/stage_01 JSON으로 현 게임 재현(하드코딩 제거).
- **M2** 맵/경로 데이터화: ASCII 그리드 + PathService(BFS) + SlotService.
- **M3** 전투 Def 주입: 적/타워/발사체가 Def에서 스탯 읽기 + 풀링. 효과 시스템 뼈대(slow/splash).
- **M4** 경제·이벤트화: Economy + GameEvents 전면(UI 구독 대비).
- **M5** **uGUI 전환 + 입력 분리**: HUD/건설 팔레트/타워 패널/웨이브 미리보기 + 설치 타일 시각화(UX 4중 처방) + 배속/일시정지. ← "사용법 모름" 해소 핵심
- **M6** 타이틀·스테이지 선택·결과(★) + ProgressStore(localStorage) + 스테이지 2~3 추가.
- **M7** 튜토리얼 스테이지(스텝 게이팅).
- **M8** 업그레이드/판매 + 오디오(Kenney) + 주스(데미지 숫자·골드 팝업·이펙트).
→ **여기까지가 "검증용 수직 슬라이스"**(튜토 + 3스테이지 + 풀 UX).

**B. 확장** — 검증 후
- 타워 5종/적 9종 완성, 월드2(설원)·월드3(크리스탈), 비행/분기경로/보스, 스테이지 4~15, 밸런싱.
- (선택) Firebase 클라우드 세이브·랭킹, 엔드리스 모드.

## 8. 리스크 / 의존성
- **모델 4종으로 적 9종 커버** → 런타임 머티리얼 틴트/스케일 코드 필수(아트 비용 0 유지의 핵심).
- uGUI 한글 → **Pretendard SDF 폰트에셋** 별도 생성 필요(TMP).
- 새 "종류"는 데이터로 추가 가능하나 새 "메커니즘"(공중적·분기경로·보호막)은 엔진 작업.
- 각 마일스톤 후 WebGL 빌드 회귀(빌드 수 분 소요) — 자동 빌드 스크립트 유지.
