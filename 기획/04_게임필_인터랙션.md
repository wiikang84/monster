## 0. 현재 코드 사실 (스펙의 출발점)

작업 착수자가 바로 손댈 지점. (파일: `Assets/Scripts/`)

| 항목 | 현재 구현 | 위치 |
|---|---|---|
| 설치 | 슬롯 클릭 = **즉시 1종 박기**(미리보기·확인 없음) | `GameManager.Update()` → `BuildTower()` |
| HUD | OnGUI(IMGUI). 골드/라이프/웨이브 박스 + 시작 버튼 | `GameManager.OnGUI()` |
| 입력 | `Input.GetMouseButtonDown(0)` + `Physics.Raycast` | `GameManager.Update()` |
| 슬롯 | `TowerSlot{ bool Occupied }` 만 보유, BoxCollider 있음 | `TowerSlot.cs`, 생성은 `BuildEnvironment()` |
| 타워 | `TowerUnit{ range, fireRate, damage, projectileSpeed }`. 타깃=최근접 고정 | `TowerUnit.cs` |
| 적 | `Enemy{ Hp, speed }`. 피격/처치 피드백 없음, 체력바 없음 | `Enemy.cs` |
| 탄 | `Projectile` 추적탄. 머즐/트레일/임팩트 없음 | `Projectile.cs` |
| 그리드 | `cell ≈ 2f`(모델 측정), range = `cell*3.5 ≈ 7유닛` | `GameManager.MeasureCell()` |
| 이벤트 | `GameEvents`(Gold/Lives/Wave/BattleEnded) **이미 존재**하나 아직 Raise 안 함 | `Core/GameEvents.cs` |
| 데이터 | `TowerDef.upgrades[]`(증분형) **이미 존재**. 단 현재 BuildTower는 하드코딩 | `Data/TowerDef.cs` |

**핵심 제약**
- WebGL: GC 스파이크 금지 → 이펙트/탄/팝업은 **오브젝트 풀링**. 파티클은 짧고(≤0.4s) 입자 적게.
- 마우스+터치 공용: 호버가 없는 터치를 1급 시민으로. "호버 미리보기"는 마우스 보조, 터치는 **탭 2단계**로.
- IMGUI는 매 프레임 즉시모드 → 패널·버튼은 OnGUI로 당장 가능. **3D 표시물(사거리 원·고스트·체력바·데미지숫자)** 은 씬 오브젝트로.

## 1. 슬롯 호버 / 선택 흐름

### 1.1 설계 결론 (관습 벤치마크 → 우리 채택)

| 관습 | 사례 | 우리 채택 |
|---|---|---|
| 빈 슬롯에 **라디얼 빌드 메뉴** | Kingdom Rush(빈 자리 클릭→4타워 부채꼴) | **채택**. 고정 슬롯과 궁합 최고. 슬롯 탭→그 자리에 미니 빌드 메뉴 |
| 카드 **드래그-드롭** 배치 | Bloons TD6, PvZ | 보조로만(데스크톱 옵션). 고정 슬롯엔 라디얼이 더 명확 |
| **즉시 설치** | (현재 우리) | 폐기. "고를 수 있다"는 키우는 맛의 1단계. 단 타워 1종 구간(튜토 St1)은 즉시설치 유지 |

> **St1(튜토)** = 타워 1종 → 슬롯 탭 = 즉시 설치(02 튜토리얼과 일치).
> **St2~** = 타워 2종 이상 → 슬롯 탭 = **라디얼 빌드 메뉴** 오픈.

### 1.2 마우스(PC) 흐름

```
빈 슬롯에 커서 진입(hover)
  → 슬롯 펄스 링 강조 + 슬롯 위 ⊕ 빌보드 밝아짐
  → (타워 1종 모드면) 고스트 타워 + 사거리 원 프리뷰 표시
  → 비용 툴팁: 커서 옆 작은 박스 "펄스 터렛 · 50골드"
커서 이탈 → 프리뷰 제거
빈 슬롯 클릭
  → 1종 모드: 즉시 설치 확정
  → 2종+ 모드: 라디얼 빌드 메뉴 오픈(슬롯 화면좌표 중심)
     · 메뉴 항목 hover → 그 타워의 사거리 원 + 고스트가 슬롯에 미리 표시(중요: 고를 때마다 사거리가 바뀜을 즉시 보여줌)
     · 항목 클릭 → 설치 확정 / 골드 부족 항목은 회색·가격 빨강
빈 곳 클릭 / 우클릭 / ESC → 메뉴 닫기·프리뷰 제거(차감 없음)
```

### 1.3 터치(모바일) 흐름 — 호버가 없으므로 2탭

```
빈 슬롯 탭(1차)
  → 라디얼 빌드 메뉴 오픈 + 그 슬롯에 기본(첫 타워) 사거리 원 미리 표시
  → 메뉴 항목 탭 → 설치 확정
  → (1종 모드면 메뉴 생략, 첫 탭에 즉시 설치 — 단 3초 [되돌리기] 토스트로 실수 방지, 02 §5)
바깥 탭 → 메뉴 닫기
```

> 마우스의 "hover 프리뷰"를 터치는 "**라디얼 항목 탭 전, 항목에 손가락 올린 상태(press-and-hold 0.15s)**"로 흉내내도 좋으나, MVP는 **항목 탭=즉시 확정**으로 단순화. 라디얼 자체가 이미 "고르는 단계"라 미리보기 부담이 작다.

### 1.4 사거리 원 3D 렌더 — **권고: LineRenderer 원**

세 방식 비교(WebGL 기준):

| 방식 | 장점 | 단점 | 판정 |
|---|---|---|---|
| **LineRenderer 원** (세그먼트 48~64개, y=바닥+0.05) | 셰이더 0개, 머티리얼 1개, 풀링 쉬움, 반경 애니메이션은 점 좌표만 갱신 | 채워진 면 아님(테두리 링) | ✅ **권고(MVP)** |
| 평면 데칼/쿼드 + 반투명 원 텍스처 | 채워진 디스크 가능, 부드러움 | 텍스처 1장 필요, URP면 데칼 셰이더 의존 | 보조(uGUI 패스에서 디스크 원하면) |
| 반투명 실린더 메시 | 즉시 | 높이감 어색, 카메라 부감서 면적 왜곡, 알파 정렬 이슈 | ✕ |

**LineRenderer 원 스펙(당장 구현)**
- `positions = 64`, `loop = true`, `useWorldSpace = false`, `widthMultiplier = 0.06 * cell`.
- 머티리얼: `Sprites/Default` 또는 unlit, 색 = 시안 `#36C5FF` 알파 0.9(빈 슬롯 프리뷰) / 노랑 `#FFD23F`(선택된 설치 타워).
- 좌표: `for i in 64: angle=i/64*2π; p=(cos·r, 0, sin·r)`, r = 타워 `range`(유닛). 바닥 살짝 띄움 y=+0.05.
- **반경 확장 애니메이션**(업그레이드 시): r을 0.25s 동안 `oldRange→newRange`로 `EaseOutCubic` 보간하며 매 프레임 64점 재계산. 동시에 링을 한 번 펄스(width ×1.6→×1.0).
- 풀: 화면에 사거리 원은 동시 최대 1~2개(프리뷰 1 + 선택 1)면 충분 → 인스턴스 2개 캐싱.

> "채워진 디스크"가 꼭 필요하면(슬롯 전체 강조용) 02의 펄스 링과 별개로 uGUI/M5에서 데칼 추가. MVP는 **테두리 링으로 충분히 명확**(킹덤러시·BTD6도 테두리 위주).

## 2. 설치된 타워 선택 → 액션 패널

### 2.1 트리거 / 상태

```
설치된 타워(또는 그 base) 클릭/탭
  → 선택 상태 진입(selectedTower = t)
  → 그 타워에 노랑 사거리 원 ON
  → 액션 패널 표시(02 §3 레이아웃)
  → (이전 선택 있으면 해제)
빈 곳/다른 타워/ESC → 해제 또는 대상 변경
```

### 2.2 패널 구성 (Kingdom Rush식 벤치마크 → IMGUI 즉시 구현)

킹덤러시는 타워 클릭 시 **두 개의 큰 업그레이드 버튼(좌우) + 판매(하단 작게) + 비용/효과를 버튼 위에 미리보기**. 우리는 슬롯 1티어=직선 업그레이드라 **단일 업그레이드 버튼**으로 단순화.

```
┌─ 펄스 터렛  Lv.1 ──────────────────────────┐
│  데미지 10   사거리 3.0   공속 1.5/s         │   ← 현재 스탯
│  ┌──────────────────────────────────────┐ │
│  │ ⬆ 업그레이드  (50골드)                  │ │   ← 다음 티어 효과 미리보기 동봉
│  │   데미지 10→16  공속 1.5→1.8           │ │      (초록 = 증가)
│  └──────────────────────────────────────┘ │
│  타게팅: [최근접▼]   [💲 판매 +35]           │   ← 토글 + 환불액
└────────────────────────────────────────────┘
```

**IMGUI 구현 형태(당장)** — `GameManager.OnGUI()` 안, `selectedTower != null` 일 때만 그림:
- 위치: 데스크톱 = 화면 좌하단 고정 박스(360×150). 모바일 = 하단 풀폭 바텀시트(02 §3).
- 헤더 `GUI.Label`: `$"{def.displayName}  Lv.{tier}"`.
- 스탯 줄 `GUI.Label`: 현재 dmg/range/fireRate.
- 업그레이드 `GUI.Button`(대형, 56px 높이):
  - 라벨 = `$"⬆ 업그레이드 ({upgradeCost}골드)"`, 그 아래 작은 줄 `$"데미지 {d}→{d2}  공속 {f}→{f2}"`.
  - **골드 부족** → `GUI.enabled=false` + 가격 빨강. **최종 티어** → "MAX" 표기·비활성.
- 타게팅 토글 `GUI.Button`: 누를 때마다 `Nearest→First→MaxHp→MinHp` 순환(라벨로 현재값 표시). `TowerUnit`에 `TargetingMode` 필드 추가 필요.
- 판매 `GUI.Button`: `$"💲 판매 +{refund}"`. 누르면 즉시 환불·타워 제거·슬롯 `Occupied=false`·먼지 퍼프.

**환불액 공식**: 누적 투자 골드 × **0.7** (03 §2.3). 누적 = 설치비 + 업그레이드비 합. → `TowerUnit.investedGold` 필드 추가, 설치/업글 때 가산.

> M5(uGUI)에서 이 박스를 Kenney 9-slice 패널 + 버튼 위 아이콘으로 승격. **레이아웃·텍스트는 동일하게 유지**(IMGUI에서 검증한 카피 그대로 이식).

### 2.3 타게팅 우선순위 (당장 넣을 최소)

`TowerUnit.Update()`의 타깃 선택을 `TargetingMode`로 분기:
- **Nearest**(현재): 최근접. (기본값)
- **First**: 경로상 가장 앞선 적(=`Enemy.wp` 큰 쪽, 동률이면 다음 웨이포인트까지 거리 작은 쪽). 누수 방지에 가장 유효 → **권장 기본값을 First로 바꾸는 것도 검토**.
- **MaxHp / MinHp**: 체력 최대/최소.

→ `Enemy`에 "경로 진행도"(현재 `wp` + 다음 점까지 잔여거리) getter 추가하면 First 구현 가능.

## 3. 업그레이드 "키우는 맛" 설계

### 3.1 티어 수 권고: **3티어** (T1 설치 + T2 + T3)

03 §2.3이 이미 3티어 + 환불 70%로 설계됨. 그대로 채택. 3티어는 "성장 체감"과 "선택 피로 없음"의 균형점(킹덤러시 4~5단계는 타워 종류가 적을 때 유효, 우리는 5종이라 종×3=15갈래로 충분).

### 3.2 티어업 순간 "쾅" 체감 4종 동시 발사

업그레이드 버튼 누르는 **그 0.4초**에 아래를 한꺼번에:

| # | 요소 | 스펙 | 구현 |
|---|---|---|---|
| 1 | **모델 변화** | base 파츠 적층 또는 스케일 | MVP: 타워 `localScale ×1.0→1.12`(T2)→`×1.25`(T3) 펀치(0.2s EaseOutBack). 정식: `tower-round-build-a/c/e` 모델 스왑(03 §2.3) |
| 2 | **사거리 원 확장** | §1.4 반경 보간 0.25s + 링 펄스 | range 증가분만큼 노랑 원이 바깥으로 퍼짐 |
| 3 | **업글 이펙트** | 타워에서 위로 빛 링/스파크 1발 | Kenney 파티클 or 간이 빌보드 스프라이트 1장, 0.3s 페이드업 |
| 4 | **사운드** | 상승음(파워업) | Kenney `interface`/`digital` 팩의 상승 톤. 판매=하강 톤 |
| (보너스) | **수치 점프 텍스트** | 타워 위 "LV.2!" 금색 0.6s 떠오름 | §4 데미지 숫자 풀 재사용 |

> "수치 점프 체감"의 핵심: 패널에서 **업글 전후를 화살표(10→16)** 로 보여주고(2.2), 실제로 적이 **눈에 띄게 빨리/세게** 죽도록 T2=×1.6, T3=×2.4(03)로 점프 폭을 크게. 미세 증가는 체감 안 됨 → 큰 폭 권장.

### 3.3 데이터 연결

`TowerDef.upgrades[]`(증분형, 이미 존재) 사용. `TowerUnit`에 `int tier` + `Apply(UpgradeDef)`:
```
tier++; damage += up.damage; range += up.range; fireRate += up.fireRate(음수); projectileSpeed += up.projectileSpeed;
investedGold += up.cost;
```
사거리 원·모델·이펙트는 Apply 직후 트리거.

## 4. 주스(Juice) / 피드백 — WebGL 성능 우선

원칙: **짧게(≤0.4s)·작게·풀링**. 모든 팝업/이펙트는 사전 생성 풀에서 꺼내 쓴다(`Instantiate/Destroy` 금지).

| 사건 | 트리거(코드 지점) | 시각 | 사운드(Kenney 팩) | 풀/주의 |
|---|---|---|---|---|
| **발사** | `TowerUnit.Shoot()` | 머즐 플래시(무기 끝 0.08s 작은 빌보드 스프라이트) + 탄 트레일(LineRenderer 짧게 or TrailRenderer) | `sci-fi` laser, **연사 시 0.1s 쿨다운·볼륨↓** | 트레일은 탄에 부착, 풀 |
| **피격** | `Enemy.TakeDamage()` | 적 머티리얼 **흰색 플래시 0.1s**(emission 펄스) + 작은 스파크 | `impact` impactSoft, 간헐 재생 | 동시 다수면 사운드 throttle |
| **데미지 숫자** | `TakeDamage()` | 적 위 작은 숫자, 0.5s 떠오르며 페이드. 치명타=금색·1.3배 | (무음) | **빌보드 텍스트풀 16개**. 기본 ON, 설정 OFF |
| **넉백(약)** | `TakeDamage()` | 진행방향 반대 0.05*cell 살짝 밀림(경로 이탈 금지=시각만) | — | 선택. 과하면 OFF |
| **처치** | `OnEnemyKilled()` | 파편 퍼프(작은 입자 6개) + 스케일 1.0→0 0.15s 후 소멸 | `sci-fi` explosion 작게 | 풀. 입자 6개 상한 |
| **골드 획득 팝업** | `OnEnemyKilled()` | 처치 위치에 "+12" 금색, 0.4s 뒤 **골드 카운터로 빨려듦**(스크린 좌표 보간) | `rpg` coin | 보상감 핵심. 텍스트풀 |
| **라이프 감소** | `OnEnemyReachedBase()` | 화면 가장자리 **빨강 비네트 1회 번쩍 0.3s** + 라이프칩 흔들림 | `interface` error/buzz | 풀스크린 1장 알파 애니 |
| **웨이브 시작** | `StartWave()` | 상단 "웨이브 N" 배너 슬라이드인 + 3·2·1 카운트 | `music-jingles` start 짧게 | |
| **웨이브 클리어** | 웨이브 종료 | "클리어 +보너스" 토스트 + 골드 팝 | jingle 짧게 | |
| **승리/패배** | `BattleEnded` | 승=별 ★ 하나씩 팝 / 패=화면 회색 페이드 | win / lose | 결과 화면 |
| **버튼·카드** | OnGUI hover/press | hover 살짝 밝게, press 눌림 | `ui` tick | 전역 |

**자원 칩 펀치**(02 §3과 일치): `GoldChanged`/`LivesChanged` 구독 → 값 변할 때 칩 스케일 1.0→1.25→1.0(0.15s) + 색 점멸(골드↑녹색 / 라이프↓빨강). IMGUI에선 박스 폰트 크기를 0.15s간 살짝 키우는 식으로 흉내 가능.

**오디오 시동**: 브라우저 autoplay 정책 → **첫 사용자 탭 이후** AudioListener/BGM 시작(02 §6).

**풀 크기 권고**(WebGL): 데미지숫자 16, 골드팝업 8, 피격스파크 12, 처치퍼프 8, 트레일=탄과 동수. 동시 적 상한(03 고밀도 웨이브)이 ~30이라 이 정도면 충분.

## 6. 구현 우선순위 — IMGUI-now vs uGUI-later

오너가 "키우는 맛·손맛"을 **가장 빨리 체감**하도록 임팩트순 정렬. (M5=uGUI 전환, M8=업그레이드·오디오·주스 — 마스터플랜 로드맵)

### 6.A IMGUI/3D로 지금 바로 (임팩트 큰 순) — "M8 일부를 앞당김"

| 순위 | 항목 | 왜 먼저 | 난이도 | 의존 |
|---|---|---|---|---|
| **1** | **설치 타워 선택 → 사거리 원 + 액션 패널(업그레이드/판매)** | "키우는 맛"의 본체. 클릭→패널이 즉시 게임을 게임답게 | 중 | TowerUnit에 tier/invested/targeting 필드 + OnGUI 패널 |
| **2** | **업그레이드 체감 4종**(모델스케일·사거리확장·이펙트·사운드) | 1과 묶여야 "성장"이 보임 | 중 | TowerDef.upgrades 연결 |
| **3** | **피격 흰색 플래시 + 처치 퍼프 + 데미지 숫자** | 손맛. 적이 "반응"하기 시작 | 하 | Enemy.TakeDamage 훅 + 풀 |
| **4** | **골드 획득 팝업 + 자원칩 펀치** | 보상 루프 체감(왜 처치가 즐거운가) | 하 | GameEvents Raise 연결 |
| **5** | **발사 머즐+트레일 / 처치·발사 사운드** | 살아있는 전장. WebGL 사운드 시동 처리 포함 | 하~중 | Audio 서비스 최소 |
| **6** | **사거리 원 프리뷰 + 슬롯 호버 펄스/⊕**(설치 전) | "어디에 뭘 짓는지" 명확(02 핵심결함) | 중 | LineRenderer 원 |
| **7** | **라이프 감소 비네트 + 웨이브 시작 배너** | 위기·리듬감 | 하 | OnGUI 풀스크린 |
| **8** | **적 체력바(피격시) + 보스 상단 바** | 진척 가독성 | 중 | 빌보드 풀 |
| **9** | **타게팅 토글** | 깊이(있으면 좋음, 급하진 않음) | 하 | First용 진행도 getter |

> 1~5만 들어가도 오너 체감 "밋밋 → 게임답다"가 확 바뀜. 이게 **이번 패스의 최소 목표**.

### 6.B uGUI(M5)로 미룰 것 — IMGUI로는 한계/품질 부족

| 항목 | 이유 |
|---|---|
| 라디얼 빌드 메뉴(부채꼴 애니·아이콘) | OnGUI로 부채꼴·아이콘·트윈이 조악. uGUI 버튼+레이아웃이 정석. **임시로 IMGUI 세로 리스트 메뉴는 가능** |
| 건설 팔레트 카드(아이콘·9-slice·골드부족 흔들림) | Kenney UI Pack 9-slice 필요 |
| 바텀시트(모바일 타워 패널) 슬라이드·터치타깃 48px | uGUI 앵커/세이프에어리어 필요 |
| 한글 고급 타이포(Pretendard SDF, 외곽선) | TMP SDF 폰트에셋 필요(IMGUI는 동적폰트로 버팀) |
| 다음 웨이브 미리보기 아이콘 그리드, 적 도감 | 레이아웃·스크롤 필요 |
| 타이틀/스테이지선택/결과(★) 화면 | M6 |
| 튜토리얼 딤 마스크 spotlight | M7 |

> **이식 원칙**: 6.A에서 IMGUI로 확정한 **카피·레이아웃·수치·트리거를 그대로** uGUI로 옮긴다. IMGUI는 "디자인 검증용 1차", uGUI는 "마감". 버린 작업 없음.

### 6.C 채택 데이터/구조 메모(착수자용)
- `TowerUnit` 필드 추가: `int tier=1`, `int investedGold`, `TargetingMode targeting`, `string defId`.
- `BuildTower`를 `TowerDef` 주입형으로(현재 하드코딩) → 라디얼 메뉴가 defId 전달.
- `GameEvents`에 게임오브젝트 인자 이벤트(EnemyHit/Killed/TowerSelected/Upgraded/Sold) 추가는 M3/M4 예정 → 그 전까지는 GameManager가 직접 풀 호출해도 됨(주스 먼저, 분리 나중).
- 풀 매니저(`FxPool`) 1개: 데미지숫자/골드팝업/스파크/퍼프/머즐 종류별 큐.

## 부록. 핵심 수치 치트시트
- 사거리 원: LineRenderer 64점, 시안 프리뷰 / 노랑 선택, 확장 0.25s EaseOutCubic.
- 업글 티어: 3티어, T2 주력×1.6·T3×2.4, 환불 누적×0.7.
- 모델 펀치 스케일: T1=1.0 / T2=1.12 / T3=1.25, 0.2s EaseOutBack.
- 이펙트 수명 ≤0.4s, 피격 플래시 0.1s, 데미지숫자 0.5s, 골드팝업 0.4s, 비네트 0.3s.
- 풀: 데미지숫자16·골드팝8·스파크12·퍼프8.
- 사운드 throttle: 발사 0.1s·피격 간헐(동시 다수 시 1회만).
- 체력바: 100%면 숨김, 손상 시 표시, 보스 상시+상단 대형.
- 타게팅: Nearest/First/MaxHp/MinHp (기본 First 검토).
