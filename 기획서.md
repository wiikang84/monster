# Tower (가제) — 게임 기획서

> 3D 저폴리 타워 디펜스 · Unity 6 · WebGL · 무료(CC0) 에셋만 사용
> 최초 작성 2026-06-22

## 1. 한 줄 컨셉
길을 따라 몰려오는 적을, 길목에 타워를 세워 막아내는 3D 저폴리 타워 디펜스.

## 2. 기본 방향 (확정 사항)
| 항목 | 결정 |
|---|---|
| 엔진 | **Unity 6 (6000.0.58f1)** — C#로 개발 (클로드가 코드 직접 작성) |
| 플랫폼 | **웹(WebGL)** — 개발은 PC 에디터에서 테스트, 완성 시 WebGL 빌드 |
| 배포 | **목업 확인 후 Firebase 진행 여부 결정** (현재 미정) |
| 조작 | 마우스 + 터치 **겸용** (클릭/탭으로 타워 설치·선택) |
| 아트 | **자체 아트 0.** Kenney Tower Defense Kit(CC0) 단일 스타일로 통일 |
| 비용 | **0원.** CC0/무료 에셋만 |

## 3. 핵심 게임 루프
1. 웨이브 시작 (플레이어가 시작 버튼 또는 자동)
2. 적이 정해진 **경로(웨이포인트)**를 따라 진입
3. 길목에 설치된 **타워가 자동으로 사거리 내 적을 조준·공격**
4. 적 처치 → **골드** 획득
5. 골드로 **타워 설치 / 업그레이드**
6. 적이 끝점(기지)에 도달하면 **라이프 감소**
7. 라이프 0 → 게임 오버 / 모든 웨이브 클리어 → 승리

## 4. MVP(목업) 범위 — "이것까지 돌면 목업 완성"
- 맵 1개, 경로 1개(웨이포인트)
- 타워 **2종** (예: 단발 화살탑 / 느린 광역탑)
- 적 **2종** (기본 / 빠름)
- 웨이브 **3개** (점점 많아짐)
- 타워 설치(슬롯/그리드) + 골드 소모
- 타워 자동 조준·발사·데미지
- 골드 / 라이프 / 웨이브 표시 UI + 시작 버튼
- 게임 오버 / 승리 화면
- ※ 1차 목업은 **그레이박스(기본 도형)**로 게임 느낌부터 확인 → 이후 Kenney 모델로 교체

## 5. 확장 아이디어 (목업 이후, 선택)
- 타워 종류 추가 / 업그레이드 단계 / 판매
- 적 종류·보스, 다중 경로, 맵 추가
- 사운드(Kenney Audio CC0), 이펙트, 점수·랭킹
- 난이도/스테이지 선택

## 6. 아트·사운드 에셋 계획 (전부 무료/CC0)
| 용도 | 에셋 | 라이선스 | 비고 |
|---|---|---|---|
| 3D 모델(타워/적/길/지형) | **Kenney Tower Defense Kit** (160개, 3D) | CC0 | kenney.nl/assets/tower-defense-kit, 표기 불필요 |
| 효과음/UI | Kenney Audio · UI Pack | CC0 | 필요 시 |
| 폰트 | Google Fonts (OFL) | OFL | 필요 시 |
> 원칙: **한 라이브러리(Kenney)로 스타일 통일.** 사실적 에셋과 섞지 않음.

## 7. 기술 설계 원칙
- **코드 우선(code-first):** 가능한 한 런타임 C# 코드로 오브젝트를 생성/배치해 **에디터 수작업을 최소화** (클로드가 파일로 작성·수정하기 쉬운 구조).
- 씬에는 빈 GameObject 1개 + 부트스트랩 스크립트만 두고 나머지는 코드로 구성하는 방향 우선.
- 마우스/터치 입력은 공통 입력 레이어로 추상화.

## 8. 개발 로드맵
- [ ] 0. Unity 프로젝트 생성 (tower/game)
- [ ] 1. 그레이박스 프로토타입: 경로 + 적 이동 + 기지 도달 시 라이프 감소
- [ ] 2. 타워 설치(슬롯) + 자동 조준·발사 + 적 처치/골드
- [ ] 3. 웨이브 매니저 + UI(골드/라이프/웨이브) + 승/패
- [ ] 4. **목업 플레이 확인 → Firebase 배포 여부 결정** ← 체크포인트
- [ ] 5. Kenney 모델 교체(외형) + 사운드
- [ ] 6. 밸런싱 / 확장
